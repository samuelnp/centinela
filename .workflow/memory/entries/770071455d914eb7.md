# team-dashboard — gatekeeper

### Gatekeeper Report

**Status:** SAFE

Feature `team-dashboard` adds a new read-only `centinela dashboard` command backed by a new pure aggregator package `internal/teamdashboard`. The change is purely additive: it reads existing sources (active workflow JSONs, roadmap.json, the telemetry event log) and reuses `insights.Gates` for ranking. It modifies no shared entity and introduces no conflict with adjacent features. Architecture boundaries (G2 layering, G7 thin-cmd) are respected, `centinela validate` passes (exit 0), and read-only purity is independently verified.

## Analyzed Specs

- `specs/team-dashboard.feature` (subject) — `centinela dashboard`: three-panel board (in-flight features, roadmap burn-down, gate health), `--json` stable contract, honest empty states, owner column via git seam, aggregator purity.
- `specs/centinela-insights.feature` — consumed read-only: dashboard delegates gate ranking to `insights.Gates`. No command/package name overlap; no shared-state mutation. The "ranking matches insights" scenario is by-design reuse, not a conflict.
- `specs/governance-telemetry.feature` — owner of the telemetry event log that dashboard reads. Dashboard is a read-only consumer (`telemetry.ReadDefault()`); it never writes telemetry. No conflict.
- Roadmap/feature-tracking specs (e.g. `roadmap-*.feature`) — dashboard reads `roadmap.Load()` and `roadmap.Summary()` (schedulable-only) read-only for the burn-down panel. No write path; no conflict.
- Swept remaining `specs/*.feature` for `centinela dashboard` / `teamdashboard` collisions: none found. The command name and package name are new and unique.

## Findings

**Conflict detection — SAFE (additive, no shared-entity modification).**
- New command `dashboard` + new package `internal/teamdashboard`; no other spec defines either name. No modification to workflow/roadmap/telemetry/insights write paths — dashboard is a strict consumer.

**G7 (no business logic in outer layer) — PASS.**
- `cmd/centinela/dashboard.go` is a thin orchestrator: it reads sources (`workflow.ActiveWorkflows`, `roadmap.Load`, `telemetry.ReadDefault`), derives owners via the `gitOwner` seam in `dashboard_owner.go`, calls `teamdashboard.Compute`, and routes output (text vs `--json`). All composition/derivation logic (age, step index, burn-down, gate tally) lives in `internal/teamdashboard`. Rendering lives in `internal/ui`. No decisions in cmd/.

**G2 (layer boundaries) — PASS.**
- `internal/teamdashboard` non-test imports are exactly `internal/workflow` (domain), `internal/roadmap` (domain), `internal/telemetry` (leaf), `internal/insights` (aggregator), plus stdlib (`time`). No `cmd/`, no `internal/ui`, no `os`, no `os/exec`, no file I/O. (The lone `"os"`/`cmd/` regex hit was a false positive inside the package doc comment.)
- Mapped into the **aggregator** layer in BOTH PROJECT.md G2 (full paragraph describing `teamdashboard → insights` as the allowed aggregator→aggregator edge) AND `centinela.toml` `[[gates.import_graph.layers]]` aggregator `paths` (`internal/teamdashboard/**`, `allow = ["domain","leaf","aggregator"]`).
- **No import cycle:** none of `workflow`/`roadmap`/`telemetry`/`insights` imports `teamdashboard` (grep-verified). `teamdashboard` is imported only by `cmd/` and (its `Dashboard` type) by `internal/ui` for rendering, mirroring the established insights pattern.

**git stays out of the pure aggregator — PASS.**
- `os/exec`/git appears only in `cmd/centinela/dashboard_owner.go` via the overridable `var gitOwner` seam (tests override it). The aggregator receives owners as a pre-built `map[string]string`; a git error resolves to `"unknown"` and never aborts the board.

**File size (G1) — PASS.** All source files ≤100 lines:
- `internal/teamdashboard`: dashboard.go 73, features.go 65, burndown.go 36, gatehealth.go 21, compute.go 17.
- `cmd/centinela`: dashboard.go 63, dashboard_owner.go 42.
- `internal/ui`: render_dashboard_panels.go 79, render_dashboard.go 29.
- G1 gate reported "All files under 100 lines."

**Read-only purity — PASS (independently verified).**
- Built `./cmd/centinela` to `/tmp` and ran `dashboard` + `dashboard --json`: both exit 0, three panels render, `--json` top-level keys are exactly `Features`/`Roadmap`/`Gates`. `.workflow` file-mtime checksum identical before/after; no new files appeared anywhere under the project. Matches the spec's read-only and stable-contract scenarios.

**`centinela validate` — PASS, exit 0.**
- G1 ✓, G-Build cross-compile ✓ (6 targets), roadmap_drift ✓; `go test ./...` ✓, acceptance ✓, coverage (95%) ✓, check-fmt ✓.
- `import_graph` and `spec-traceability-gate` show ⚠ warnings — these are the documented pre-existing non-failing kinds ("packages match no configured layer" / "scenarios without acceptance coverage"); full-repo `validate --full` also exits 0 and neither warning mentions team-dashboard. Not a team-dashboard regression.
