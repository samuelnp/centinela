#### Prior Finding 1 — spec annotation format — RESOLVED
The spec now asserts the shipped per-runner inline format. AC1/AC6 assert
`big-thinker (model: claude-opus-4-7 (claude)` plus `model: anthropic/claude-opus-4-7 (opencode)`;
AC2/AC3 assert `documentation-specialist (model: claude-haiku-4-5-20251001 (claude)` and
`feature-specialist (model: claude-sonnet-4-6 (claude)`. These match what
`orchestration_annotate.go` emits via `resolvedPerRunner` (`fmt.Sprintf("model: %s (%s)", id, key)`
joined over `AllowedRunnerKeys()`), and match the green acceptance assertions in
`configurable_subagent_models_test.go`. No surviving `(model: <tier>)`-only annotations.

#### Prior Finding 2 — integration helper re-implementation — RESOLVED
`annotateRolesHelper` in `configurable_subagent_models_integration_test.go` no longer
re-implements the old format. It now iterates `orchestration.AllowedRunnerKeys()` and calls
the real domain resolver `orchestration.ResolveModel(role, models, modelMap, Runner(key))`
per runner — byte-for-byte mirroring `orchestration_annotate.go`. It consumes models/modelMap
from the shared `routingFromConfig(cfg)` (defined once in
`configurable_model_routing_integration_test.go`), which maps real `config` accessors onto
domain types. Assertions match real output (fast→`claude-haiku-4-5-20251001`, balanced default
→`claude-sonnet-4-6`).

#### New-conflict re-checks
- **Old-format `(model: reasoning|balanced|fast)` assertions:** grep over `specs/ tests/`
  returned exactly one hit — `configurable_model_routing_test.go:55` asserting
  `model: reasoning (codex)`. This is NOT the stale format: it is AC7 of the *other* feature,
  verifying codex (which has no model-map entry) falls back to the bare tier name (Step 4 of
  `ResolveModel`, ok=false) rather than leaking another runner's concrete ID. Confirmed against
  `model_routing.go` and `models.go` (`AllowedRunnerKeys` returns claude/opencode/codex; no
  built-in codex tier model). Legitimate production behavior — not a finding.
- **3-arg `ResolveModel(` calls:** none. Every call site (cmd, internal, tests) passes the
  4th `Runner` argument matching the current signature.
- **Duplicate declarations / compile issues:** none. `routingFromConfig`/`loadAt` are declared
  once (model_routing integration file); `annotateRolesHelper` once (subagent_models integration
  file). They share `package integration_test` cleanly — the subagent file reuses, not redefines,
  `routingFromConfig`.
- **Build/vet/tests:** `go vet ./...` clean. `go test ./...` → 1109 passed across 24 packages.
  Feature tiers (integration + acceptance + unit) → 322 passed.

#### Recommendation
