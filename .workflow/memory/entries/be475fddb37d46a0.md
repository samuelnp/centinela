- **Colocated-only coverage rule**: The 95% gate is driven exclusively by colocated `*_test.go` files; files under `tests/` do not contribute to the per-package coverage percentage. Integration test `TestNewTestFiles_WithinG1Limit` walks the relevant dirs and asserts G1 for every pattern added by this feature, ensuring the rule is not bypassed by a large test file smuggled in colocated.
- **G1 line-count enforcement on test files**: All 55 new `*_test.go` files are verified ≤100 lines each by the integration test. Files that approach the limit (e.g., split across `cov2_*` and `_more_*` siblings) are individually checked — the walk covers all naming patterns.
- **Deferred un-unit-testable paths**: `runMcpServe`/`mcpConnectSelf` (blocks waiting for an MCP connection), `WriteBytesAtomic` low-level I/O error branches (require fault injection at the syscall level), and `runVulnTool` external-binary seam are deliberately excluded from unit coverage rather than covered with hollow or assertion-free tests. Each is recorded in the Backlog phase of `.workflow/roadmap.json` with a descriptive summary. The acceptance test `TestDeferredPaths_InRoadmapBacklog` asserts all three slugs are present.
- **Parallel-round duplicate-declaration risk**: The code step authored tests for 9 packages in parallel rounds. After all rounds, `go vet ./...` was run over the full module to detect any duplicate function/variable declarations introduced when two rounds added similarly-named helpers to the same package. The vet was clean. Future parallel rounds must repeat this verification.
- **Gate floor unchanged**: `TestCoverageGate_FloorUnchanged` (integration) and `TestCoverageGate_ScriptAndFloor` (acceptance) both assert that `scripts/check-coverage.sh` still contains `MIN_COVERAGE:-95.0` — the feature raises coverage headroom above the floor rather than adjusting the gate itself.
- **MCP server loop remains uncovered**: `runMcpServe` and `mcpConnectSelf` cannot be exercised by a fast-running unit test without an in-memory MCP transport (not yet wired). Deferred to roadmap item `unit-test-mcp-server-in-memory-transport`.
- **Atomic-write syscall faults unexercised**: The `WriteBytesAtomic` rename-error and sync-error branches require OS-level fault injection (e.g., overlayfs read-only remount). Deferred to `fault-inject-atomic-write-error-paths`.
- **External vuln-tool seam not stubbed**: `runVulnTool` shells out to an external binary; the timeout/error path requires either a real binary or a test-seam extraction. Deferred to `unit-test-vuln-tool-external-seam`.
- **Unreachable `json.Marshal` / embedded-FS errors**: A small set of `json.Marshal` calls on known-good structs and `fs.ReadFile` calls on embedded assets have error branches that are structurally unreachable at runtime. These are intentionally excluded — forcing coverage via reflection hacks would be hollow.
