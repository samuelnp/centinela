- **Env authoritative over config.** `CENTINELA_HEADLESS=1`/`true` forces headless ON even when `[headless] enabled=false`; `0`/`false` forces it OFF even when `enabled=true` and `detect_ci=true`. The env value, when non-empty, short-circuits both config tiers.
- **detect_ci opt-in is back-compat.** `detect_ci=true` + `CI=true` → headless; `detect_ci=false` + `CI=true` → NOT headless (a CI run that did not opt in keeps interactive behavior). `detect_ci=true` with `CI` absent → not headless.
- **Zero-config default off.** No env, `enabled=false`, `detect_ci=false` → not headless; byte-identical to interactive behavior. A nil `*Config` also resolves to not-headless once the env signal is absent.
- **Empty env falls through.** An empty `CENTINELA_HEADLESS` string is treated as unset and falls through to the config + detect_ci tiers (does not force OFF).
- **Headless wins over every_step.** `shouldRenderReviewPrompt` short-circuits to `false` under `IsHeadless` BEFORE the per-knob resolver, so an explicit `step_confirmation_mode=every_step` is overridden. With headless off, every_step still renders (back-compat).
- **Plan-advisor silent under headless.** `runHookPlanAdvisor` returns `nil` before `loadActiveWorkflows()` when headless, emitting nothing even on a plan-step workflow whose brief would trigger advisor questions. Headless off → directives emit as before.
- **Verdict fail still emits JSON.** `runVerdict` writes the full JSON packet to stdout BEFORE returning the silenced sentinel error, so a failing run is still machine-consumable. The packet always marshals, even with an empty evidence index.
- **Deterministic golden output.** `AssembleVerdict` injects `Now` and uses no maps (Counts is a struct); evidence is pre-sorted by role. Two runs over fixed inputs produce byte-identical indented JSON matching the golden file.
- **Gate lowercase vs verify uppercase.** Gate statuses map to lowercase (`pass`/`fail`/`warn`/`skip`; unknown → `unknown`); verify statuses keep native UPPERCASE (`PASS`/`FAIL`/`CONFIG-ERROR`/…) verbatim.
- **Empty evidence index.** With no on-disk `.workflow/<feature>-<role>.json` files, `EvidenceIndex` returns an empty NON-nil slice so the packet emits a JSON array, not `null`.
- **Warnings non-failing.** A WARN verify check (or warn gate) is tallied in the summary counts but never flips the verdict; fail iff `!gates.AllPassed || verify.HasFailures`.
- **Full-scan in v1.** The verdict always runs `gates.RunAll` (nil filter), equivalent to a full scan; it never uses a changed-only diff filter.
- **Missing workflow.** `runVerdict` returns the underlying `workflow.Load` error before assembling a packet when the feature has no workflow JSON; callers get a non-sentinel error and no packet. Acceptance coverage exercises the assemble path with an explicit workflow rather than the load failure.
- **Run-info error paths.** `config.Load`/`workflow.Load` failure branches in the command are not exercised by golden tests (they need a corrupt project on disk); they surface as plain errors before any JSON is written.
- **detect_ci precedence is intentionally last.** A future third signal added below detect_ci must preserve env > enabled > detect_ci ordering or the truth table breaks; the table test pins all twelve combinations.
