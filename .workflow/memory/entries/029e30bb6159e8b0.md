### Gatekeeper Report: adoption-baseline
**Date:** 2026-06-24
**Status:** SAFE

#### Analyzed Specs
- `specs/adoption-baseline.feature` (the new feature under review)
- `specs/audit-baseline-ratchet.feature` (the sibling that owns the shared baseline file and `centinela audit` / `audit baseline`)
- PROJECT.md → Gatekeeper Paths + G2 layer-boundary rule

#### Findings

**1. Shared baseline file coexistence with `audit baseline` — No conflict.**
- Affected spec / scenario: `audit-baseline-ratchet.feature` → all "Recording a baseline" and "Determinism" scenarios; `adoption-baseline.feature` → "The baseline written by adopt is byte-identical to audit.Record plus audit.Save output".
- Risk: NONE. Both `cmd/centinela/audit_baseline.go::runAuditBaseline` and `internal/audit/adopt.go::Adopt` write the SAME path (`cfg.Gates.AuditBaseline.BaselinePath`, default `.workflow/audit-baseline.json`) via the identical pair `audit.Record(cfg)` + `audit.Save(path, b)`. `Save` canonicalizes (sortBaseline) and emits the same `{scheme, version, gates}` shape regardless of caller. A baseline produced by `adopt` is therefore byte-identical to one produced by `audit baseline` on the same unchanged repo, and is fully readable by `centinela audit`'s ratchet (`Load` + fingerprint partitioning). No format divergence, no second artifact, no second file path.
- Suggestion: none.

**2. `Baseline.Total()` and `Adopt` are purely additive — No conflict.**
- Affected spec / scenario: none.
- Risk: NONE. `grep` of `internal/` + `cmd/` (excluding tests) shows `Baseline.Total()` is called only from the two NEW files (`internal/ui/render_adopt.go`, `cmd/centinela/adopt_render.go`); `audit.Adopt` / `RenderAdoption` are called only from the new `adopt` command. No existing function body, gate, fingerprint routine, baseline format, or call site was modified. `Total()` is a new read-only method on `Baseline` (it does not touch `Record`, `Save`, `Load`, `ratchet`, or the existing `countFingerprints` helper, which remains independent).
- Suggestion: none.

**3. Behavioral drift in audit-baseline-ratchet scenarios — None.**
- Affected spec / scenario: every scenario in `audit-baseline-ratchet.feature`.
- Risk: NONE. `centinela audit` and `centinela audit baseline` are untouched (no edits to `cmd/centinela/audit.go`, `audit_baseline.go`, `internal/audit/ratchet.go`, `record.go`, `fingerprint.go`, `participation.go`). `adopt` is a brand-new sibling top-level command; the only semantic deltas (skip-if-exists default, `--force`, the adoption report, `--json` verdict) live entirely in new code and do not feed back into the ratchet path. The ratchet's safe-adoption default and "only tightens" guarantees are unaffected.
- Suggestion: none.

**4. Command-name collision — None.**
- Affected spec / scenario: n/a (CLI surface).
- Risk: NONE. `adopt` is registered as a NEW top-level command via `rootCmd.AddCommand(adoptCmd)`. The existing audit baseline command is the SUBcommand `baseline` under `auditCmd` (`Use: "audit"` → `Use: "baseline"`), invoked as `centinela audit baseline`. There is no pre-existing `adopt` command or subcommand anywhere in `cmd/centinela/`. No collision.
- Suggestion: none.

**5. G2 layer boundaries — Respected.**
- Affected spec / scenario: n/a (architecture).
- Risk: NONE. `cmd/centinela/adopt*.go` import `internal/audit`, `internal/config`, `internal/ui` (cmd → internal is allowed). `internal/ui/render_adopt.go` imports `internal/audit` for the `audit.Outcome` rendering type (read-only, mirrors the existing `internal/ui` read-only allowance for domain rendering types). `internal/audit/adopt.go` imports only `internal/config` (leaf), consistent with the rest of `internal/audit`. The skip-if-exists business rule lives in `internal/audit` (domain), not in `cmd/` — correct per separation of concerns. No new cross-layer or cyclic import.
- Suggestion: none.

none

#### Recommendation
SAFE to proceed to the validation step. The feature is strictly additive: it composes the already-shipped `Record`/`Save`/`Load` machinery onto the shared `.workflow/audit-baseline.json`, writing a byte-identical file that the existing ratchet reads unchanged. No existing function, gate, fingerprint scheme, baseline format, or command was modified; `adopt` does not collide with `audit baseline`; and no `audit-baseline-ratchet.feature` scenario changes behavior. `go build ./...` succeeds and `go test ./internal/audit/... ./internal/ui/... ./cmd/centinela/...` passes (423 tests).
