#### Findings

**SyncKind collapse (SyncClaudeHooks + SyncOpenCodePlug -> SyncKindPrewriteHook)**

- **Affected spec:** specs/opencode-hook-parity.feature
- **Affected scenario:** Plugin invokes prewrite and postwrite around file edits
- **Risk assessed:** The refactor collapsed two distinct SyncKind constants into one (`SyncKindPrewriteHook`), relying on path-based dispatch inside `applyItem()` to route Claude hooks vs. OpenCode plugin writes. A mistake in that path check could silently apply the wrong write function.
- **Finding:** No conflict. `applyItem()` correctly checks `it.Path == pluginFile` to route the OpenCode plugin write; any other path falls through to `InjectHooks` (Claude). The `golden_parity_test.go` locks byte-for-byte output of both agents against pre-refactor golden fixtures, making this the regression tripwire. No code anywhere references the old constant names — the only mention is in the `.workflow/` senior-engineer documentation. The opencode-hook-parity scenarios are fully protected.

**G2 layer rule compliance**

- **Affected spec:** specs/g2-import-graph-gate.feature
- **Risk assessed:** New adapter files in `internal/setup/` could import `cmd/` or `internal/ui/`, violating the G2 upward-dependency rule.
- **Finding:** No violation. `internal/setup/` imports only the Go standard library. The cmd layer (`init_agent.go`, `migrate_setup.go`) correctly imports `internal/setup` and `internal/ui` downward. The UI-rendering switch in `runHarnessSetup()` is an outer-layer concern and is correctly located in `cmd/centinela/init_agent.go`.

**Aider --agent value extending validation**

- **Affected spec:** specs/adapt-opencode-support.feature, specs/migrate-full-sync.feature
- **Risk assessed:** Adding "aider" to IsValidAgent could break existing tests that assume a closed set of valid agents.
- **Finding:** No conflict. `IsValidAgent` is now fully registry-driven. All existing tests for "claude", "opencode", "both" still pass — "aider" is additive. The test in `cmd/centinela/helpers_test.go` explicitly asserts all four names are valid and "nope" is invalid.

**G1 file size gate**

- **Risk assessed:** New files could exceed the 100-line limit.
- **Finding:** All new and modified files are within bounds. Largest file is `opencode_plugin.go` at exactly 100 lines. No G1 exception is needed.

**Spec traceability — 7 scenarios without acceptance-tier // Scenario: comments**

- **Affected spec:** specs/host-harness-adapters.feature (AC4 byte-parity x2; AC2 structural x2; AGENTS.md shared-surface; migrate-idempotent-scoped; partial-install)
- **Risk assessed:** spec_traceability gate is severity=warn; missing acceptance comments may trigger a warning.
- **Finding:** Acceptable coverage at appropriate tier. The 7 non-acceptance-tagged scenarios break down as:
  - AC4 byte-parity (x2): covered by `internal/setup/golden_parity_test.go` — a byte-level regression test is more precise than an acceptance test for this concern.
  - AC2 structural no-if-ladder + "both" no-special-case (x2): covered by `tests/unit/host_harness_adapters_registry_unit_test.go` — structural/source-inspection assertions belong at unit tier.
  - AGENTS.md shared-surface: guaranteed by `planManagedFile()` idempotency (already-correct file returns nil item); covered implicitly by the idempotency unit test.
  - migrate-idempotent-scoped: covered by `cmd/centinela/migrate_setup_test.go` integration tests.
  - partial-install (adding Aider leaves Claude files untouched): covered by `cmd/centinela/init_aider_test.go`.
  All 7 scenarios have real test assertions at an appropriate tier. The spec_traceability gate will warn but not block, and the warning is justified.
