governance over MCP (official Go SDK v1.2.0, stdio). Reuses
`verdict.AssembleVerdict`; advisory-only (allow/warn/block); shim maps block→exit 2.

- [x] All source + test files ≤100 lines (G1 green; `mcp_shim.go` split into a client file to stay under).
- [x] No cross-layer import violations: `internal/mcp` joins the aggregator layer, importing verdict/gates/verify/workflow/config + the external SDK; imported only by `cmd/`. SDK types confined to `internal/mcp` + 3 cmd files. `import_graph` warning empty-body (benign).

- [x] No business logic in the outer layer: cmd/ wires; logic in `internal/mcp` + reused engines.
- [x] i18n: house-style tool descriptions; no new locale surface.
- [x] Advisory contract: server only reads; the shim (not the server) performs the deny. Verified by `TestAccMcpShimBlockAndAllow`.
- [x] New dependency pinned: `modelcontextprotocol/go-sdk v1.2.0` (+ go.sum); isolated for one-package upgrades.

#### Findings
- `spec-traceability-gate` + `import_graph` warnings are empty-body, non-blocking,

  clean error or no-op.
