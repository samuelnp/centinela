#### Findings
- Reindex invariant (a): correct. `insertPhaseAt`/`removePhaseAt` (internal/roadmap/rawphase_struct.go) shift `d.dirty` keys >= pos by +1 (insert) or drop-at-idx/-1-above-idx (remove), in lockstep with the `d.phases` slice splice — both structures are mutated together in the same function, so `render()` (internal/roadmap/rawrender.go) indexing `d.dirty[i]`/`d.phases[i]` by position stays consistent. Directly proven byte-exact by `TestReindex_InsertEarlierWhileLaterDirty` and `TestReindex_RemoveMiddleWhileLaterDirty` (internal/roadmap/rawphase_struct_test.go), which dirty a later phase's feature then perform the structural op and assert the full rendered output string.
- Note byte-identity (b): correct. `rawPhase.Note` is `json:"note,omitempty"` (rawphase_struct.go) so a note-less phase encodes with no `note` key. The untouched-phase render path (`indentValue` in rawrender.go) re-indents original bytes rather than re-encoding via the struct, so phases never touched by an add/rename never pass through the struct at all. Only actually-mutated phases re-encode (rename, or the dirty-phase render path `renderDirtyPhase`), and `renderDirtyPhase` only emits a `"note"` key when present in the raw map (`if v, ok := phase[key]; ok`). No output change for note-less phases.
- Reserved-name guard coverage (c): correct. All three entry points (`PhaseAdd`, `PhaseRename` on both old and new names, `PhaseRemove`) reject via the shared `isNonSchedulablePhase` predicate (Backlog ∪ Baseline), matching existing defer/promote usage sites (mutate_shared.go, rawmove.go, roadmap.go, readiness.go, view.go). `insertPosition` deliberately allows Backlog as an `--after` anchor (position only, not membership) per spec — correctly distinct from the reserved-name identity guard.
- `--force` prune consistency (d): correct. `forceRemovePhase` (phase_remove_force.go) removes the phase in-memory, decodes to `*Roadmap`, runs `ValidateDependencies` before any disk write, and only writes `roadmap.json` if that passes; artifact pruning (`removeFeatureEntries`) runs after the roadmap write, and `revalidateArtifacts` (`ValidateAnalysis`+`ValidateQuality`) re-confirms the final on-disk state. A surviving cross-phase `dependsOn` is refused with all three files (roadmap/analysis/quality) byte-identical, proven by `TestPhaseRemove_ForceRefusedOnSurvivingDep`; the success path is proven by `TestPhaseRemove_ForcePrunesAndValidates`.
- Layering / file size / i18n (e): none. All 15 changed/added files are 15-91 lines (well under the 100-line G1 cap). Imports respect the n-tier graph: `cmd/centinela/roadmap_phase*.go` imports only `internal/roadmap` + `internal/ui` (outer layer, no business logic — each RunE is a thin flag-parse-and-delegate); `internal/roadmap/*` imports only stdlib + intra-package. i18n gate is disabled project-wide (`gates.i18n = false`, English-only CLI per PROJECT.md → Locales) so the new English error strings are compliant, not a violation.
- Full suite: `go test ./internal/roadmap/... ./cmd/centinela/... ./tests/...` — 1839 tests passed, 0 failed.
