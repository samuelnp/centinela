# New Project Setup Guide

This guide explains how to bring **Centinela** into a new project.

## What to Copy

Copy these files/directories from any project that uses this framework:

```
scripts/centinela-workflow.sh
CLAUDE.md
PROJECT.md.template
docs/architecture/
```

Do NOT copy `PROJECT.md` — that file is project-specific and will be generated by the wizard.

## Step 1: Run the Setup Wizard

Open a Claude Code session in the new project root and run:

```
/centinela-setup
```

The wizard guides you through 5 phases of questions (~10 minutes):
1. **Concept** — elevator pitch and problem statement
2. **Architecture** — pattern choice and rationale
3. **Tech Stack** — language, framework, persistence, testing, i18n
4. **Domain** — core entities and external integrations
5. **Confirmation** — review and correct before writing

At the end, the wizard writes `PROJECT.md` automatically.

## Step 2: Verify the Setup

After the wizard completes, confirm:

```bash
# PROJECT.md exists and looks complete
cat PROJECT.md

# Workflow script is executable
chmod +x scripts/centinela-workflow.sh
scripts/centinela-workflow.sh status-all
```

## Step 3: Start Your First Feature

```bash
scripts/centinela-workflow.sh start <first-feature-name>
```

This will fail with a helpful error if `PROJECT.md` is missing.

## Step 4: Follow the 4-Step Workflow

```
plan → code → tests → validate
```

See `CLAUDE.md` for the full workflow documentation.

## Step 2b: Create `scripts/validate.sh` (required)

`workflow.sh` calls `scripts/validate.sh` during the `validate` step. This file is **mandatory** — the workflow will refuse to complete without it.

It must run lint + type/static analysis + full test suite for your stack:

```bash
#!/bin/bash
set -euo pipefail
# Example for Ruby on Rails:
bundle exec rubocop --no-color
bundle exec rspec --format progress
```

```bash
chmod +x scripts/validate.sh
```

The `/centinela-setup` wizard creates this file for you based on your tech stack answers.

## What Each File Does

| File | Purpose |
|------|---------|
| `CLAUDE.md` | Framework rules: how to build (architecture, workflow, naming) |
| `PROJECT.md` | Project definition: what to build (domain, stack, structure) |
| `PROJECT.md.template` | Blank template for manual filling (alternative to wizard) |
| `scripts/centinela-workflow.sh` | Enforces the 4-step development workflow |
| `scripts/validate.sh` | **Required:** runs lint + type check + tests for your stack |
| `docs/architecture/` | Architecture reference documentation |

## Filling PROJECT.md Manually

If you prefer to fill in `PROJECT.md` without the wizard:

1. Copy `PROJECT.md.template` to `PROJECT.md`
2. Follow the `<!-- comment -->` instructions in each section
3. Every section is required — do not leave placeholders in place
4. Pay special attention to **Gatekeeper Paths** — wrong paths break gatekeeper checks

## Verification Checklist

Before starting your first feature, confirm:

- [ ] `PROJECT.md` exists and has no `<!-- comment -->` placeholders
- [ ] All locales listed in PROJECT.md → Locales have message files at the path specified
- [ ] `scripts/centinela-workflow.sh start test-feature` runs without error (then delete the workflow file)
- [ ] The gatekeeper paths in PROJECT.md match the actual folder structure
