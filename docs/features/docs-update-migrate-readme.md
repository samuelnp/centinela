# docs-update-migrate-readme

## Problem

The README still emphasizes `init` for setup changes, while migration now supports
full-sync updates for docs and setup artifacts.

## User Stories

- As a user, I want README guidance for `centinela migrate` full-sync behavior.
- As a user, I want scoped setup migration examples with `--agent`.

## Acceptance Criteria

- README documents `centinela migrate` preview and `--apply` full sync.
- README includes `centinela migrate setup --agent ...` usage.
- README hook section reflects migration guidance beyond docs-only updates.
