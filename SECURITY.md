# Security Policy

## Supported Versions

Centinela is pre-1.0. Only the latest tagged release on `main` receives security fixes.

| Version            | Supported |
|--------------------|-----------|
| Latest tag on main | ✅        |
| Older releases     | ❌        |

## Reporting a Vulnerability

**Do not file public GitHub issues for security problems.**

Preferred channel:

- Open a private GitHub Security Advisory: <https://github.com/samuelnp/centinela/security/advisories/new>

Alternative:

- Email **sanapo@gmail.com** with the subject line `[centinela security]`. Include reproduction steps, affected version, and impact assessment.

## What to expect

- **Acknowledgement**: within 72 hours.
- **Initial triage**: within 7 days, including severity assessment.
- **Fix or mitigation plan**: communicated before public disclosure.
- **Public disclosure**: coordinated with the reporter once a fix is available, typically within 90 days of report.

Reporters acting in good faith will be credited in release notes unless they request otherwise.
